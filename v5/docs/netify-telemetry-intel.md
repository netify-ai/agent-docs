# Intelligence Telemetry - Netify Agent v5

> [!TIP]
> This is an AI-optimized version of our documentation. 
> For the full human experience, including interactive examples and branding, 
> view the **[Intelligence Telemetry Guide](https://www.netify.ai/documentation/agent/v5/integrations/telemetry/intelligence)** on our website.

The `flow_intel` telemetry payload provides network intelligence indicators generated by the Netify Intelligence plugin. It maps explainable risk indicators to observed network flows.

## Risk Scoring
Risks are scored from 0 to 100. They are categorized to highlight threats such as anomalous TLS certificates, tor relays, or cleartext credentials.

## Format Types
Configured via the `flow_metadata` property in the Intelligence Processor:
1.  **Full (`full`)**: Emits the complete flow record alongside the `intel` array.
2.  **Minimalist (`none`)**: Emits only the flow `digest` and the `intel` array. Best when flows are already logged elsewhere.
3.  **Stats (`stats`)**: Emits the flow `digest`, the `intel` array, and summarized flow statistics (bytes, packets) up to the point of detection.

## The `intel` Array
Regardless of the format type, all intelligence records include an `intel` array containing the specific detections:
*   **category**: The intelligence category tag (e.g., `tls_certificate_audit`).
*   **indicator**: The configured tag of the specific indicator that triggered.
*   **indicator_driver**: The underlying driver powering the detection.
*   **score**: The risk score (0-100).
*   **event**: The detection lifecycle event that triggered it (e.g., `dpi_complete`).
*   **criteria**: The resolved criteria values used during evaluation.
